# Image2 UI Design Prompt: 收租吧

请为一款微信小程序「收租吧」设计一套高级、克制、专业的移动端产品 UI。目标用户是个人房东/小型公寓经营者，管理约 80 间房，需要高效处理房源、房间、租约、租客、月度账单、收款、收据、月度经营导出和日常提醒。请严格按照以下功能和交互逻辑设计，不要做营销落地页。

## 设计目标

生成一张高保真 UI 设计稿展示板，包含 7-8 个关键手机屏幕，尺寸按微信小程序常见移动端比例设计，例如 390x844。整体质感要高级、安静、清晰、适合高频运营使用，像专业的房东经营后台，而不是花哨消费级 App。

## 必须包含的屏幕

1. 首页工作台
   - 展示今日经营概览：本月应收、本月已收、逾期金额、空置房间、15 天内到期租约。
   - 展示待处理提醒列表：租约到期、逾期未收、押金异常、水电录入提醒。
   - 首页只做概览和提醒，不放复杂收据管理。

2. 业务维护页
   - 作为底部 tab 的第二项，入口包含：月度经营导出、收据记录、房源维护、租客维护、提醒设置。
   - 这是偏工具型页面，信息密度要高，但分组清楚。

3. 房源/房间列表页
   - 按房源展示房间，支持状态标签：已租、空置、即将到期、逾期。
   - 房间卡片展示房间名、租客、月租、下次应收日期、当前状态。

4. 房间/租约详情页
   - 展示当前房间、租客、租约起止日期、押金、月租、账单状态。
   - 按月份展示账单组，每个月包含房租、押金、管理费、水费、电费等明细。
   - 收据入口必须是月份级别：按钮文案为「开具本租约本月收据」或「查看本月收据」。
   - 禁止在每个费用细项上放单独的收据按钮。
   - 费用细项只展示项目名称、应收日期、实收日期、金额、状态。
   - 房东支出/业主费用要和租客应收账单分区展示，收据不得包含房东支出。

5. 月度经营导出页
   - 有月份选择、范围选择：全部 / 按房源 / 按房间。
   - 展示导出前摘要：应收、实收、欠费、业主费用、净收入。
   - 有导出记录列表，支持查看、重新导出、删除记录。

6. 收据记录页
   - 从业务维护页进入。
   - 顶部有「按租约开收据」区域：选择租约、选择月份、点击「开具该租约本月收据」。
   - 已开收据按月份自动分组，例如「2026 年 4 月」，每组展示收据数量和合计金额。
   - 支持筛选：月份、房源、房间、租客。
   - 收据卡片展示：收据编号、房源/房间、租客、收款日期、金额、账单数。
   - 卡片操作：查看收据、回到房间、删除。
   - 删除是唯一纠错动作，需要表现为危险操作。不要设计作废、重开、状态筛选。

7. 收据详情页
   - 正式凭证标题：「收款收据（非发票）」。
   - 内容包含：收据编号、房源/房间、租客、收款日期、收款项目明细、合计金额、备注、生成时间。
   - 收款项目明细用表格展示：项目、应收日期、实收日期、金额。
   - 不能出现「收款人」字段。
   - 操作按钮：导出 PDF 打印版、复制收据摘要、分享凭证、删除收据。
   - 收据应当看起来适合打印，日期和金额列宽稳定，不重叠。

8. 我的/设置页
   - 展示基础账号信息、数据备份提示、云函数部署状态/同步提示可以做成简洁管理入口。

## 关键业务规则，必须体现在设计里

- 收据按「租约 + 月份」开具，同一个租约同一个月只能有一张有效收款收据。
- 收据是月度账单整体凭证，不是每个费用细项的凭证。
- 已开收据可以查看和删除。删除后该租约月份可以重新开具。
- 不设计作废、重开、作废原因、已作废状态、状态筛选。
- 不设计收款人字段。
- 不展示任何内部 ID，例如 receiptId、billId、leaseId、roomId。
- 收据记录要能自动整理，优先按月份分组，也能按房源/房间/租客筛选。
- 月度经营导出和收据记录都放在业务维护页，不要塞到首页。

## 视觉风格

- 高级、冷静、专业、可长期使用，偏现代经营工具。
- 主色使用墨黑、暖白、浅灰、深靛蓝或低饱和绿色作为辅助色；危险操作用克制红色。
- 不要紫色渐变，不要大面积蓝紫，不要营销 hero，不要装饰性光斑/球体/气泡。
- 卡片圆角不超过 8px，层级清楚，不要卡片套卡片。
- 使用清晰中文信息层级，金额和日期用等宽或表格数字风格。
- 按钮尽量搭配图标：导出、删除、分享、复制、筛选、返回房间。
- 界面要信息密度足够，适合房东反复查看、筛选、处理，而不是只好看。

## 输出要求

请输出一张完整 UI 设计展示图，包含多个手机屏幕并排展示，屏幕之间标注页面名称。视觉要像成熟产品设计稿，组件状态完整，包括空状态、危险删除确认、筛选状态、已开收据状态、PDF 导出动作。所有文案使用中文，严格遵守以上业务规则。
