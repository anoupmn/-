# 需求文档：收租吧

**定义日期：** 2026-04-01  
**核心价值：** 让我每天打开小程序就能立刻知道该收谁的租、哪间房有异常、下一步该处理什么。

## v1 需求

### 登录与身份

- [x] **AUTH-01**：用户可以通过微信登录，作为 v1 的唯一房东账号使用系统
- [x] **AUTH-02**：用户在同一设备上重新打开小程序后，登录状态可以持续保留

### 房源与房间模型

- [ ] **ASST-01**：用户可以创建和编辑一个房源，用来表示可出租资产
- [ ] **ASST-02**：用户可以在房源下创建和编辑房间，以支持分房出租场景
- [ ] **ASST-03**：用户可以标记某个可出租单元是整租模式还是分房出租模式
- [ ] **ASST-04**：用户可以查看每个可出租单元的当前状态，包括已出租、空置、即将到期、已逾期或异常

### 租户与租约

- [ ] **LEASE-01**：用户可以创建和编辑租户基础档案信息
- [ ] **LEASE-02**：用户可以创建租约，并将租约关联到房源或房间以及对应租户
- [ ] **LEASE-03**：用户可以为每个租约单独配置租金周期、起止日期等规则
- [ ] **LEASE-04**：用户可以结束某个租约，且不会丢失历史租户与历史租约记录
- [ ] **LEASE-05**：用户可以查看每个房间或可出租单元的完整历任租户记录

### 分项账单

- [ ] **BILL-01**：用户可以记录房租、押金、水费、电费、物业费和其他杂费等分项费用
- [ ] **BILL-02**：用户可以查看每一笔分项费用的状态，包括待收、已收或逾期
- [ ] **BILL-03**：用户可以查看每个有效租约的下一笔应收金额和应收日期
- [ ] **BILL-04**：用户可以在租约内分别查看房租类费用和非房租类费用

### 首页概览

- [ ] **DASH-01**：用户可以在首页看到 15 天内到期的可出租单元数量
- [ ] **DASH-02**：用户可以在首页看到当前空置的可出租单元数量
- [ ] **DASH-03**：用户可以在首页看到异常房屋或房间列表
- [ ] **DASH-04**：用户可以从首页概览项进入对应的可操作筛选列表

### 提醒与建议

- [ ] **ALRT-01**：当租约接近应收日期时，用户会收到应用内提醒
- [ ] **ALRT-02**：当费用变成逾期状态时，用户会收到应用内提醒
- [ ] **ALRT-03**：当房屋或房间空置超过设定阈值时，用户会收到应用内提醒
- [ ] **ALRT-04**：用户可以手动将某个房屋或房间标记为异常，并在提醒和列表中看到该标记
- [ ] **ALRT-05**：系统可以根据到期、逾期、空置或维修异常等规则，给出下一步应处理什么的规则型建议

### 微信提醒

- [ ] **NOTF-01**：用户可以订阅微信官方提醒消息，用于接收 v1 的关键提醒
- [ ] **NOTF-02**：用户订阅后，可以收到房租到期、逾期或房屋异常等微信提醒消息
- [ ] **NOTF-03**：用户可以控制哪些提醒规则启用微信自我提醒

### 维修记录与统计

- [ ] **REPR-01**：用户可以为房源或房间创建一条维修记录，并填写固定维修分类和备注
- [ ] **REPR-02**：在适用情况下，用户可以将维修记录关联到对应租户的租住期间
- [ ] **REPR-03**：用户可以查看每套房或每个房间的累计维修次数
- [ ] **REPR-04**：用户可以查看每套房或每个房间在每任租户期间的维修次数
- [ ] **REPR-05**：用户可以查看每套房或每个房间的主要维修类别
- [ ] **REPR-06**：当维修频次超过规则阈值时，系统可以将房屋或房间标记为异常

### 列表与详情页

- [ ] **LIST-01**：用户可以浏览所有房屋和房间的列表，并看到关键信息与状态
- [ ] **LIST-02**：用户可以进入房屋或房间详情页，查看当前租约、租户、账单、提醒和维修信息
- [ ] **LIST-03**：用户可以在详情页查看历史租户和历史维修记录

### 数据录入与导入

- [ ] **IMPT-01**：用户无需依赖导入功能，也可以手动录入房屋、房间、租户、租约等基础经营数据；分项费用录入属于第 2 阶段账单能力
- [ ] **IMPT-02**：用户可以通过约定的 Excel 模板导入房屋、房间、租户、租约和费用数据
- [ ] **IMPT-03**：用户在正式导入前可以预览 Excel 数据校验结果
- [ ] **IMPT-04**：当 Excel 数据有问题时，用户可以看到按行展示的校验错误信息

## v2 需求

### 提醒扩展

- **NOTF-04**：用户可以新增家人或合伙人作为提醒接收人
- **NOTF-05**：用户可以为不同接收人配置不同提醒规则

### 附件与档案

- **DOCS-01**：用户可以为租约上传和查看合同附件
- **DOCS-02**：用户可以上传维修照片或交接记录

### 统计报表

- **RPT-01**：用户可以查看按月汇总的收租、空置和维修趋势报表
- **RPT-02**：用户可以导出经营相关报表

### 租客侧能力

- **TNNT-01**：租客可以收到直接提醒，或进入自助流程
- **TNNT-02**：租客可以通过独立入口提交报修请求

## 明确不做

| 功能 | 原因 |
|------|------|
| 租客端小程序 | v1 明确只做房东自用 |
| 多房东账号系统 | 当前范围只有一个个人房东账号 |
| 复杂角色权限体系 | 在真实多人协作出现前没有必要 |
| AI 自动定价或自动催收策略 | v1 先做可解释的规则型建议 |
| 高保真定制 UI 设计 | v1 先以功能闭环为主 |
