# 需求文档：收租吧

**定义日期：** 2026-04-01
**最后更新：** 2026-04-27
**核心价值：** 让我每天打开小程序就能立刻知道该收谁的租、哪间房有异常、下一步该处理什么。

## v1 需求

### 登录与身份

- [x] **AUTH-01**：用户可以通过微信登录，作为 v1 的唯一房东账号使用系统
- [x] **AUTH-02**：用户在同一设备上重新打开小程序后，登录状态可以持续保留

### 房源与房间模型

- [x] **ASST-01**：用户可以创建和编辑一个房源，用来表示可出租资产
- [x] **ASST-02**：用户可以在房源下创建和编辑房间，以支持分房出租场景
- [x] **ASST-03**：用户可以标记某个可出租单元是整租模式还是分房出租模式
- [x] **ASST-04**：用户可以查看每个可出租单元的当前状态，包括已出租、空置、即将到期、已逾期或异常

### 租户与租约

- [x] **LEASE-01**：用户可以创建和编辑租户基础档案信息
- [x] **LEASE-02**：用户可以创建租约，并将租约关联到房源或房间以及对应租户
- [x] **LEASE-03**：用户可以为每个租约单独配置租金周期、起止日期等规则
- [x] **LEASE-04**：用户可以结束某个租约，且不会丢失历史租户与历史租约记录
- [x] **LEASE-05**：用户可以查看每个房间或可出租单元的完整历任租户记录

### 分项账单

- [x] **BILL-01**：用户可以记录房租、押金、水费、电费、物业费和其他杂费等分项费用
- [x] **BILL-02**：用户可以查看每一笔分项费用的状态，包括待收、已收或逾期
- [x] **BILL-03**：用户可以查看每个有效租约的下一笔应收金额和应收日期
- [x] **BILL-04**：用户可以在租约内分别查看房租类费用和非房租类费用

### 首页概览

- [x] **DASH-01**：用户可以在首页看到 15 天内到期的可出租单元数量
- [x] **DASH-02**：用户可以在首页看到当前空置的可出租单元数量
- [x] **DASH-03**：用户可以在首页看到异常房屋或房间列表
- [x] **DASH-04**：用户可以从首页概览项进入对应的可操作筛选列表

### 提醒与建议

- [x] **ALRT-01**：当租约接近应收日期时，用户会收到应用内提醒
- [x] **ALRT-02**：当费用变成逾期状态时，用户会收到应用内提醒
- [x] **ALRT-03**：当房屋或房间空置超过设定阈值时，用户会收到应用内提醒
- [x] **ALRT-04**：用户可以手动将某个房屋或房间标记为异常，并在提醒和列表中看到该标记
- [x] **ALRT-05**：系统可以根据到期、逾期、空置或维修异常等规则，给出下一步应处理什么的规则型建议

### 微信提醒

- [x] **NOTF-01**：用户可以订阅微信官方提醒消息，用于接收 v1 的关键提醒
- [x] **NOTF-03**：用户可以控制哪些提醒规则启用微信自我提醒

> **已删除范围：** 微信官方提醒消息实际发送闭环不再纳入当前路线。

### 维修记录与统计

- [x] **REPR-01**：用户可以为房源或房间创建一条维修记录，并填写固定维修分类和备注
- [x] **REPR-02**：在适用情况下，用户可以将维修记录关联到对应租户的租住期间
- [x] **REPR-03**：用户可以查看每套房或每个房间的累计维修次数
- [x] **REPR-04**：用户可以查看每套房或每个房间在每任租户期间的维修次数
- [x] **REPR-05**：用户可以查看每套房或每个房间的主要维修类别
- [x] **REPR-06**：当维修频次超过规则阈值时，系统可以将房屋或房间标记为异常
- [x] **OPEX-01**：维修、保洁、打理、请人管理等房东支出可以独立记录，金额可选，录入目的为留痕、复盘和导出
- [x] **OPEX-02**：维修类支出参与房间问题分析；保洁、打理、请人管理等支出不计入维修高频异常

### 列表与详情页

- [x] **LIST-01**：用户可以浏览所有房屋和房间的列表，并看到关键信息与状态
- [x] **LIST-02**：用户可以进入房屋或房间详情页，查看当前租约、租户、账单、提醒和维修信息
- [x] **LIST-03**：用户可以在详情页查看历史租户和历史维修记录

### 数据录入

- [x] **IMPT-01**：用户无需依赖导入功能，也可以手动录入房屋、房间、租户、租约等基础经营数据；分项费用录入属于第 2 阶段账单能力

> **已删除范围：** Excel 批量导入房屋、房间、租户、租约和费用数据不再纳入当前路线。

### 费用与账单模型优化

- [x] **FEE-01**：租约录入固定内置费用：租金、押金、管理费、消防押金、锁卡押金
- [x] **FEE-02**：消防押金和锁卡押金作为一次性押金类应收，进入账单、收据和导出，但不进入逾期提醒或下一笔租金核心提示
- [x] **FEE-03**：管理费默认周期性费用，但允许在租约中改为一次性费用
- [x] **FEE-04**：自定义费用必须选择性质：周期性费用、一次性费用或押金类费用
- [x] **UTIL-01**：水电费录入采用上期抄数、本期抄数和单价，由服务端自动计算用量与金额
- [x] **UTIL-02**：水电补录时默认带出同房间同类型上一笔读数和单价，并允许备注特殊情况
- [x] **CORR-01**：编辑租约时只替换未收款的系统生成账单，已收账单和手工账单不得被静默删除
- [x] **CORR-02**：租约支持安全删除；一旦产生已收账单、收据或维修/支出关联，则禁止硬删除，只能编辑、更正、结束或作废

### 月度导出与收据

- [x] **EXPT-01**：用户可以按月份导出全部房源，也可以筛选单个房源/楼栋或房间导出
- [x] **EXPT-02**：导出文件为一个 Excel 多 sheet，至少包括月度明细、账单明细、房东支出明细、退租支出明细
- [x] **EXPT-03**：月度明细参考“每月房租、水电收入及各项支出”表头，一行一个房间，展示水电读数、用量、费用、房租、维修费、其他费用、退租支出和房租水电合计
- [x] **RCPT-01**：收据只针对租客已收款项生成，不包含房东支出，并固定标注“收款收据（非发票）”
- [x] **RCPT-02**：收据内容快照固定，后续账单更正不能改写旧收据；录错时只能作废重开

## 未来可再评估

### 提醒扩展

- **NOTF-04**：用户可以新增家人或合伙人作为提醒接收人
- **NOTF-05**：用户可以为不同接收人配置不同提醒规则

### 附件与档案

- **DOCS-01**：用户可以为租约上传和查看合同附件
- **DOCS-02**：用户可以上传维修照片或交接记录

### 租客侧能力

- **TNNT-01**：租客可以收到直接提醒，或进入自助流程
- **TNNT-02**：租客可以通过独立入口提交报修请求

## 明确不做

| 功能 | 原因 |
|------|------|
| Excel 批量导入 | 本轮明确删除，当前优先手动录入、账单口径稳定和月度导出 |
| 微信官方提醒消息实际发送闭环 | 本轮明确删除，当前保留应用内提醒与订阅设置 |
| 租客端小程序 | v1 明确只做房东自用 |
| 多房东账号系统 | 当前范围只有一个个人房东账号 |
| 复杂角色权限体系 | 在真实多人协作出现前没有必要 |
| AI 自动定价或自动催收策略 | v1 先做可解释的规则型建议 |
| 高保真定制 UI 设计 | v1 先以功能闭环为主 |
