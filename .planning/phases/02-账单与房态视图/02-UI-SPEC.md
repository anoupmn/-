## Screen Contract

### Primary Screens
- `房屋列表`：继续作为经营入口，卡片列表第一屏必须可快速扫读主状态、风险标签、当前租户、下一笔应收和逾期/空置提示。
- `房屋/房间详情`：第一屏改为经营摘要优先，不允许先展示历史区块。

### Focal Point
- 详情页第一视觉焦点必须是“经营摘要卡”，不是标题，不是历史列表。
- 摘要卡必须同时容纳：主状态、风险标签、当前租户、下一笔应收日期、下一笔应收金额、逾期提示。

### Information Hierarchy
1. 经营摘要卡
2. 高频动作区：`登记收款`、`查看全部账单`
3. 当前租约区
4. 费用区：`房租`、`押金`、`非房租类费用`
5. 历史租约 / 历任租户（默认折叠）

### Interaction Rules
- 列表页卡片点击进入详情，不新增平行详情入口。
- 风险标签必须使用文案标签，不允许仅靠颜色表达。
- 历史区默认折叠，使用带文字的展开按钮；不使用纯图标交互。
- 对于 destructive 行为，按钮必须和收款类主动作视觉区分。

## Component Contract

### List Card
- 每张房屋卡片必须显示：名称、主状态、风险标签、当前租户、下一笔应收日期、下一笔应收金额。
- `逾期天数` 和 `空置天数` 属于次级提示，只在对应情形出现，不默认占位显示 `0`。

### Summary Card
- 顶部一行：单元名称 + 主状态标签。
- 第二行：风险标签组，允许同时展示多个标签。
- 第三行：当前租户。
- 第四行：下一笔应收日期与金额，金额必须明显大于普通正文层级。
- 第五行：逾期提示或“15 天内到期”提示，仅出现其一或并列出现，按风险优先级排序。

### Fee Sections
- 详情页费用区固定分成三块：`房租`、`押金`、`非房租类费用`。
- 追加费用项全部归入 `非房租类费用`。
- 每条费用记录至少显示：费用类型、应收日期、状态、金额、收款日期（已收时）。

### Historical Sections
- 历史租约与历任租户默认折叠，展开后按时间顺序展示。
- 历史区标题必须明确带“历史”字样，避免与当前经营信息混淆。

---

## State Presentation

### Main Status
- 只允许：`已出租`、`待入住`、`空置`。
- 主状态使用实底标签，统一放在摘要卡和列表卡片上方。

### Risk Tags
- 只允许：`即将到期`、`已逾期`、`异常`。
- `异常` 在 Phase 2 仅由逾期触发，不单独新增人工标记入口。
- `即将到期` 的阈值固定为 15 天内。

### Fee Status
- 只允许：`待收`、`今日到期`、`已收`、`逾期`。
- `今日到期` 需要单独标签，不与 `待收` 混写。
- 押金不显示 `逾期` 标签。
