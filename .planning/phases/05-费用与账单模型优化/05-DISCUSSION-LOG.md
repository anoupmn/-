## 维修/支出边界

| Option | Description | Selected |
|--------|-------------|----------|
| 维修费用进入租客账单 | 维修费作为补录费用向租客应收 | |
| 维修/打理作为房东支出 | 不进入租客账单，只做留痕、复盘、问题分析和导出 | ✓ |
| 维修和打理完全分开入口 | 维修和保洁/打理分别录入 | |

**User's choice:** 维修、损坏、保洁、请人打理等费用都是房东承担，录入目的只是记录和复盘，后续需要导出。  
**Notes:** 入口可合并为“记录维修/支出”，金额可选；维修类参与问题房间分析，保洁/打理不参与高频维修异常。

## 录错与删除

| Option | Description | Selected |
|--------|-------------|----------|
| 所有租约都允许删除 | 操作自由，但会破坏已收账单、维修/支出和收据历史 | |
| 租约安全删除 | 仅无已收账单、无收据、无维修/支出关联时允许删除 | ✓ |
| 完全禁止删除租约 | 历史最稳，但录错补救成本高 | |

**User's choice:** 按推荐，但确认租约录入错误时希望可以删除。  
**Notes:** 锁定“安全删除”：无经营事实时可删除，并级联删除未收款账单；已有收款、收据、维修/支出关联后禁止硬删除，只能编辑、更正、结束或作废。
