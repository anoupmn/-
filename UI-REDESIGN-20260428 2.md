# 收租吧 UI 重设计方案

生成日期：2026-04-28  
设计阶段：v1 收尾前 UI 统一调整  
参考视觉稿：`UI-REDESIGN-IMAGE2.png`  
可浏览原型：`UI-REDESIGN-PROTOTYPE.html`

## 设计目标

收租吧已经完成房源、租约、账单、提醒、维修、导出和收据闭环。收尾前 UI 调整的目标不是做营销感首页，而是把整个小程序统一成一个适合每天反复使用的经营工具：

- 首页回答“今天先处理什么”。
- 业务维护回答“我要录入、导出或维护什么”。
- 房源列表回答“哪间房现在是什么状态”。
- 房间详情回答“这间房本月要收什么、已收什么、要补什么”。
- 我的页只承接账号、订阅和设置，不混入日常业务动作。

## 设计原则

1. 高密度但不拥挤  
   主要信息一屏内尽量可扫读，减少大面积说明文字和装饰性模块。

2. 状态先于装饰  
   逾期、即将到期、空置、人工异常、已收、未收必须有稳定颜色和固定位置。

3. 操作贴近业务对象  
   收款、生成收据、补录费用、记录维修应出现在房间详情和账单上下文中，不孤立成入口堆叠。

4. 不暴露内部概念  
   所有选择都使用房源、房间、租客、月份等业务语言，不出现内部 ID。

5. 文案少而准  
   页面不解释功能怎么用，只显示当前状态、对象名称、金额、日期和明确动作。

### 字号

| 层级 | rpx | 场景 |
|------|-----|------|
| 页面标题 | 40-44 | 首页、业务维护、列表、我的 |
| 区块标题 | 32-34 | 卡片标题、分组标题 |
| 对象名称 | 30 | 房间名、租客名、账单月份 |
| 正文 | 28 | 字段值、按钮 |

| 标签 | 22 | 状态 chip、风险 tag |

### 组件

- 页面容器：`page-shell` 统一 24rpx 内边距。
- 卡片：用于独立对象或独立功能区，避免卡片套卡片。
- 列表项：使用轻边框和浅底，不再每条都做厚阴影。
- 按钮：主操作蓝色，次操作浅灰，危险操作红色。
- 标签：状态标签固定为圆角胶囊，但只用于短状态文本。
- 金额：金额统一右对齐或在摘要区突出，不和说明文案混排。

## 信息架构

### Tab 结构

| Tab | 角色 | 不应该承载 |
|-----|------|------------|
| 首页 | 经营盘面、风险优先级、今日动作 | 大量维护入口 |
| 业务维护 | 建档、租约、导出、收据辅助入口 | 经营风险长列表 |
| 房源列表 | 房间状态扫描、筛选、进入详情 | 表单编辑 |
| 我的 | 账号、订阅、设置、退出 | 日常收租动作 |

## 页面重设计

### 1. 登录页

目标：让用户快速进入工作空间。

布局：
- 顶部显示产品名“收租吧”和一句价值文案。
- 中部显示登录卡片，包含微信登录按钮、当前环境状态。
- 底部显示轻量说明：数据按当前微信账号隔离。

交互：
- 未登录访问其他页面统一 `reLaunch` 到登录页。
- 登录成功后进入首页。
- 登录失败只显示一条错误反馈，不展开技术错误。

### 2. 首页

目标：一眼知道今天先做什么。

布局：
- 顶部：问候 + 当前经营状态短句。
- KPI 横向区：已逾期、即将到期、空置过久、人工异常。
- 优先提醒：最多 3-5 条，按风险排序。
- 今日建议：一条主动作，按钮直达对应列表或详情。

交互：
- 点 KPI 进入房源列表并带临时筛选。
- 点提醒行进入对应房间详情。
- “查看全部”进入提醒中心。
- 下拉刷新重新聚合首页数据。

### 3. 提醒中心

目标：处理所有异常，而不是浏览通知。

布局：
- 顶部筛选：全部、已逾期、即将到期、空置、人工异常。
- 分组列表：每组显示数量和最重要条目。
- 每条提醒显示房源/房间、租客、原因、建议动作。

交互：
- 点提醒进入房间详情。
- 人工异常可在详情内解除或补充说明。
- 筛选切换不跳页，只更新列表。

### 4. 业务维护

目标：所有非首页的维护动作集中，但按业务任务分组。

布局：
- 房源与房间：房源维护、房源列表。
- 租客与租约：租约录入、租约列表、租户维护。
- 经营凭证：月度经营导出、收据记录入口。
- 设置类动作不放这里，留给我的页。

交互：
- 每个分组只保留 1 个主按钮和必要次按钮。
- 月度导出入口显示最近一次导出月份和范围。
- 不再出现测试数据重置入口。

### 5. 房源列表

目标：快速扫出每间房的状态和下一笔钱。

布局：
- 顶部搜索框：房源、房间、租客、状态关键词。
- 筛选 chips：全部、逾期、到期、空置、异常、已租。
- 房间行：房源/房间、租客、主状态、风险标签、下一笔应收日期、金额。
- 长列表默认显示前 12 条，搜索时显示全部匹配。

交互：
- 点房间行进入详情。
- 手动点 Tab 时清空临时筛选。
- 从首页 KPI 进入时自动应用对应筛选。

### 6. 房间详情

目标：围绕单个房间完成收租、补录、收据、维修和历史追溯。

布局：
- 顶部摘要：房源/房间、租客、主状态、下一笔应收金额。
- 当前租约：租期、租金、押金、固定费用、续租按钮。
- 月度账单：年份折叠，当前年份展开；月份折叠，当前月份展开。
- 账单项：费用名、日期、应收/实收、状态、操作。
- 维修/支出：记录入口 + 最近支出摘要。
- 历史：历史租约、历史维修、维修统计。

交互：
- 未收账单显示“登记缴费”。
- 已收账单显示“生成收据”或“查看收据”。
- 手工补录账单可删除，系统账单不可直接删除。
- 续租使用弹窗，不跳出详情页。
- 补录费用和维修/支出使用底部弹窗或页面弹层，提交后局部刷新详情。

### 7. 租约录入

目标：减少录错，明确费用性质。

布局：
- 步骤 1：选择房源和房间。
- 步骤 2：选择租客或新建租客。
- 步骤 3：租期和租金。
- 步骤 4：固定费用，内置五项 + 自定义费用。

交互：
- 房间必须通过房源筛选选择。
- 自定义费用必须选择周期性、一次性或押金类。
- 保存前展示费用摘要，避免一次性费用误认为周期费用。

### 8. 月度经营导出

目标：不用懂内部数据，也能导出正确范围并管理历史文件。

布局：
- 顶部：月份 picker。
- 范围：全部房源、单个房源、单个房间。
- 单个房源/房间使用 picker，不使用 ID 输入。
- 生成结果：文件名、房间数、账单数、收入合计、支出合计。
- 导出记录：月份、范围、生成时间、摘要、打开、删除。

交互：
- 切换为单房间时，先选房源，再选房间。
- 生成后自动刷新导出记录。
- 删除记录先二次确认，并尝试删除云文件。
- 同月不同范围文件名必须唯一。

### 9. 收据页

目标：像一张清楚的收款凭证，而不是普通详情页。

布局：
- 顶部：收款收据（非发票）+ 作废状态。
- 基本信息：编号、房源/房间、租客、收款日期、收款人。
- 明细：项目、应收日期、读数、实收金额。
- 底部：合计金额、生成时间、备注、作废原因。
- 操作：作废收据、重开收据。

交互：
- 作废必须填写原因。
- 作废后原收据保留，只显示重开入口。
- 重开生成新编号，旧收据保持作废状态。

### 10. 我的

目标：账号和订阅管理，不干扰日常业务。

布局：
- 账号卡片：昵称、账号标识。
- 订阅状态：当前授权、启用规则。
- 提醒设置入口。
- 退出登录。

交互：
- 发起/重新授权只处理订阅授权。
- 提醒设置进入二级页。
- 退出登录二次确认。

## 关键流程

### 每日收租流程

1. 打开首页。
2. 查看 KPI 和今日建议。
3. 点逾期/到期进入筛选后的房源列表。
4. 进入房间详情。
5. 登记缴费。
6. 已收后生成或查看收据。

### 录入新租约流程

1. 业务维护 -> 租约录入。
2. 选择房源/房间。
3. 选择或新建租客。
4. 填租期、租金和固定费用。
5. 保存后返回房间详情或租约列表。

### 月度导出流程

1. 业务维护 -> 月度经营导出。
2. 选择月份和范围。
3. 生成导出。
4. 打开 Excel 核对。
5. 不需要的历史记录可删除。

### 收据作废重开流程

1. 房间详情 -> 查看收据。
2. 作废收据并填写原因。
3. 重开收据。
4. 新收据进入账单关联，旧收据保留作废状态。

## 落地顺序

1. 统一全局 token：颜色、按钮、卡片、标签、列表行。
2. 先改首页、房源列表、房间详情三条最高频链路。
3. 再改业务维护、月度导出、收据页。
4. 最后整理表单页和我的页。
5. 每改一批跑页面静态测试、云函数测试和 typecheck。
