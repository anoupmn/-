# 收租吧使用说明（当前版本）

适用版本：当前仓库代码状态（更新于 2026-04-07）

## 1. 产品定位

收租吧用于个人房东做日常租赁管理，核心流程是：
登录 -> 房源维护 -> 租约录入 -> 房源列表处理账单/维修 -> 首页与提醒中心看风险。

## 2. 第一次使用（建议按这个顺序）

### 第 1 步：登录

1. 打开小程序进入登录页。
2. 点击“微信一键登录”。
3. 同意昵称授权后进入首页。
4. 如果拒绝授权，页面会提示“需要同意微信昵称授权后才能登录”。

说明：当前系统以微信登录身份隔离数据，不同用户应看到各自清单。

### 第 2 步：录入房源和房间

入口：`业务维护` Tab -> `房源维护`

1. 在“新增房源”填写出租方式、房源名称、地区、详细地址。
2. 点“保存房源”。
3. 在“已录入房源”里找到目标房源。
4. 点击右侧“+”图标进入房间维护。
5. 在房间维护页保存房间。

说明：
- 右侧垃圾桶图标是删除房源，会连带删除关联房间、租约和账单。
- 每条房源会显示“最近租金”和“最近管理费”（来自最近一次录入租约）。

### 第 3 步：录入租约

入口：`业务维护` Tab -> `租约维护` -> `租约录入`

1. 先在“选择房源与房间”里搜索并点击房源卡片（卡片变蓝表示选中）。
2. 再选择房间。
3. 填写租户姓名、租期和固定费用。
4. 点击“保存租约”。
5. 保存成功后会弹窗：
   - “继续录入”：留在当前页继续下一条。
   - “退出页面”：返回上一页。

关键校验：
- 未选房源/房间、未填租户姓名会阻止保存。
- 租约开始日期晚于结束日期会阻止保存。
- 与已有租约时间冲突会阻止保存（必须调整后再保存）。

### 第 4 步：查看在租清单并进入单元详情

入口：`业务维护` Tab -> `租约维护` -> `租约列表`

1. 页面标题为“在租房屋列表”。
2. 点击任意房源卡片进入“单元详情”。

### 第 5 步：在单元详情做日常操作

#### 5.1 月度账单

1. 展开某个月份。
2. 对待收/逾期账单点“登记缴费”。
3. 输入实际收款金额，点“确认收款”。

说明：
- 若实收金额与应收不一致，会显示小字提示“实收 ¥X，与应收不一致”。
- 补录费用产生的记录支持“删除记录”（仅补录记录可删）。

#### 5.2 补录本月费用

1. 在月度账单中点“补录本月费用”。
2. 选择费用类型并填金额。
3. 若选择“其他费用”，必须填写费用名称。
4. 点击“确认补录”。

#### 5.3 维修记录

1. 点“记录维修”。
2. 选择分类、日期，填写备注。
3. 点“确认保存”。

说明：备注必填；缺少房间信息或登录失效时会保存失败。

#### 5.4 结束租约

1. 手动提前结束：点击“结束租约”并确认。
2. 到期结束：当最后一期账单已登记收款且已到期，会弹窗提示“是否立即按期满结束租约处理”。

历史租约展示口径：
- 原租约：创建租约时约定的起止日期。
- 实际租约：实际结束日期。
- 若提前结束，会显示“提前结束租约”备注。

## 3. 四个 Tab 如何用

### 首页

看经营概览、异常摘要、今日建议，适合每天先看风险优先级。

### 业务维护

做录入与维护动作的主入口：房源维护、租约录入、租约列表。

### 房源列表

按房源/租户/风险标签搜索，点卡片进入单元详情直接处理。

### 我的

看当前账号、提醒授权状态、提醒规则入口、退出登录。

## 4. 提醒相关

1. 提醒中心按四类分组：即将到期、已逾期、空置过久、人工异常。
2. 提醒设置页只控制“提醒类型开关”，不处理阈值与日志。
3. 未完成订阅授权也能先使用业务功能。

## 5. 常见问题排查（体验测试重点）

### 5.1 提示“云函数未部署/FunctionName parameter could not be found”

原因：调用到的云函数未上传。

处理：在微信开发者工具中，把对应云函数重新上传部署后重试。

### 5.2 提示“账单不存在或无权限”

常见于账单已被删除或数据不同步。

处理：下拉刷新列表后重试；若仍失败，检查该账单是否确实存在。

### 5.3 提示“登录状态已失效，请重新登录”

处理：退出登录后重新登录。

### 5.4 维修记录保存失败

优先检查：
1. 是否填写维修备注。
2. 是否选择发生日期。
3. 当前页面是否来自有效房间详情。
4. 相关维修云函数是否已部署。

## 6. 建议的日常使用节奏

1. 每天先看首页和提醒中心，确定优先处理项。
2. 再去房源列表/单元详情处理缴费、补录、维修。
3. 新签约或换租时在业务维护里补齐房源和租约。
4. 每周检查一次提醒设置和订阅状态。
